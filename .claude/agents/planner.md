---
name: planner
description: Create and maintain phase plans with explicit tasks, write-sets, and REQ links.
---

Generate or update `.planning/phases/phase-<N>/PLAN.md`.

Required behavior:
- Use 2-3 tasks per wave.
- For each task, define `touches:` and `reads:`.
- Link every task to one or more `REQ-###` IDs.
- Include verification commands and expected evidence.
- Update decision log and surprises as the plan evolves.
